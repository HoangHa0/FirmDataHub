## 📂 1. PROJECT DIRECTORY STRUCTURE
The project is organized following a standard ETL pipeline architecture:

```text
TEAM_[5]_FirmDataHub/
│
├── sql/
│   └── schema_and_seed.sql      # SQL script to create the Database, Tables (DIM, FACT), Constraints, and Master View. Includes seed data.
│
├── etl/
│   ├── db_config.py             # Centralized database connection configuration.
│   ├── import_firms.py          # Loads the company catalog into the dim_firm table.
│   ├── create_snapshot.py       # Initializes or reuses Snapshot IDs for version control.
│   ├── import_panel.py          # Reads the Excel file and loads the 38 variables into the 6 FACT tables.
│   ├── qc_checks.py             # Scans for data quality errors based on predefined QC rules and generates a report.
│   └── export_panel.py          # Extracts the clean dataset (latest snapshots) to a CSV file for analysis.
│
├── data/
│   ├── team_tickers.csv         # The finalized list of the 20 assigned stock tickers.
│   ├── firms.xlsx               # The metadata/catalog of the 20 companies.
│   └── panel_2020_2024.xlsx     # The 5-year panel data file containing 38 variables.
│
├── outputs/
│   ├── qc_report.csv            # Data quality report (Auto-generated by qc_checks.py).
│   └── panel_latest.csv         # The final clean dataset (Auto-generated by export_panel.py).
│
├── requirements.txt             # List of required Python dependencies.
└── README.md                    # Project documentation and execution guide.
```

## 🛠️ 2. ENVIRONMENT SETUP (CRITICAL FOR REPRODUCIBILITY)
To ensure this project runs perfectly without conflicting with your global Python environment, we highly recommend using a Virtual Environment.

1. **⚙️ Prerequisites:** Ensure **MySQL Server (8.0+)** and **Python (3.8+)** are installed.
2. **🌱 Create and Activate a Virtual Environment:**
   Open your Terminal, navigate to the project's root folder, and run:
   * **For Windows:**
     ```bash
     python -m venv venv
     venv\Scripts\activate
     ```
   * **For MacOS/Linux:**
     ```bash
     python3 -m venv venv
     source venv/bin/activate
     ```
3. **📦 Install Required Libraries:**
   Once the virtual environment is activated, install the dependencies by running:
   ```bash
   pip install -r requirements.txt
   ```
   *(Core libraries used: pandas, sqlalchemy, mysql-connector-python, openpyxl)*
3. **🔑 Database Configuration:**
   Before running the scripts, please open the `etl/db_config.py` file and update the MySQL connection credentials (`DB_USER`, `DB_PASS`, `DB_HOST`) just **once** to match your local environment. All other ETL scripts will automatically inherit this connection.

## 🚀 3. HOW TO RUN THE CODE (REPRODUCIBLE PIPELINE)
To ensure the system runs smoothly and generates the correct outputs, please execute the following steps in exact order:

**🏗️ Step 1: Database Initialization**
- Open MySQL Workbench.
- Execute the entire `sql/schema_and_seed.sql` script. This will create the `midterm_vn_firm_panel` database, all necessary DIM and FACT tables, constraints, and the `vw_firm_panel_latest` Master View.

**💻 Step 2: Execute the ETL Pipeline (Via Terminal)**
Open your Terminal, navigate to the `etl/` directory (`cd etl`), and run these scripts sequentially:
1. `python import_firms.py` -> Imports the company list into the database.
2. `python import_panel.py` -> Automatically generates snapshot IDs and loads the 5-year panel data into the 6 FACT tables.
3. `python qc_checks.py` -> Runs quality control checks and outputs the `outputs/qc_report.csv` file.
4. `python export_panel.py` -> Filters for the newest versions and exports the clean dataset to `outputs/panel_latest.csv`.

**✅ Step 3: Verify the View (Optional)**
In MySQL Workbench, you can run `SELECT * FROM vw_firm_panel_latest;` to directly query the flattened, 38-variable clean dataset from the database.

## 🌐 4. DATA SOURCES & MANUAL PROCESSING
*(Note: This section addresses the data sourcing requirements outlined in the project instructions)*

To ensure high data quality and completeness across all 38 variables, our team compiled a **Consolidated Team Dataset** by extracting information from multiple reliable sources:

- **🧑‍💼 Management Reports:** Used exclusively to extract data for *Managerial/Inside Ownership*.
- **📰 Annual Reports:** Manually reviewed to extract *Process Innovation* and *Product Innovation* (assigned as 0/1 dummy variables), as well as the remaining ownership metrics (*State, Institutional, and Foreign Ownership*).
- **📈 Vietstock Platform:** Scraped/exported to accurately capture the *Market Value of Equity* and other relevant market indicators.
- **🏛️ Published Financial Statements:** Served as the primary source for the remaining variables, including all core financial metrics, cost structures, and cash flow data.

**🧽 Data Cleansing Pipeline:**
Empty cells or invalid characters (e.g., '-') within the raw datasets were automatically cleansed and converted into standard `NULL` or precise `float` values by our Python ETL pipeline before being inserted into the MySQL database.
